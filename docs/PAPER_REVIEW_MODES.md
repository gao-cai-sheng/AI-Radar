# 论文解读模式设计文档
> Created: 2023-12-23
> Purpose: 记录两种论文解读模式的设计，便于后续扩展

## 概述

系统支持两种论文解读模式，服务于不同的阅读目的：

| 模式 | 定位 | 核心问题 | 适用场景 |
|------|------|----------|----------|
| **Referee Report** | 评审视角 | "这篇论文值不值得发表？" | 综述写作、领域调研、快速筛选 |
| **Technical Learning** | 学习视角 | "我能从这篇论文学到什么？值不值得复现？" | 深度学习、工程落地、技能提升 |

---

## 模式 1: Referee Report (已实现)

**目标用户**: 需要快速评估论文价值的研究者、审稿人

**核心结构**:
- Meta Tags (任务类型、贡献类型)
- Contribution (问题清晰度、创新度)
- Methodology (技术路线、假设)
- Limitations (风险、反例)
- Reproducibility (NeurIPS Checklist)
- Verdict (5 维评分)

---

## 模式 2: Technical Learning (待实现)

**目标用户**: 想要深入学习和复现论文的工程师/研究者

**设计理念**:
- 不是"评判"，而是"学习"
- 关注"我能学到什么"而非"这篇好不好"
- 强调可操作性：具体技巧、复现建议、工程注意事项

### System Prompt (v1.0)

```markdown
你是一名在机器学习与大模型方向有多年经验的研究员和代码实现者。
我会提供一篇论文（标题、摘要、全文或 PDF）。
你的任务不是写新闻式综述，而是帮我**从技术细节出发搞懂这篇 paper，并评估是否值得我亲自复现或在工程中采用**。

回答必须结构化，严格使用以下模块和问题，并尽量引用论文中的具体细节（公式、实验设定、超参、图表编号等），不要给空泛的评价。
如果某个问题论文中没有给出信息，请明确标「论文未说明」，不要编造。
```

### 输出结构

#### 1. 贡献定位 (Contribution)
- 最小问题描述 + 自我定位 (1-2 句)
- 主要贡献列表 + 贡献类型标签
- 与 2-3 篇相关工作的**实质差异**

#### 2. 方法细节 (Method & Assumptions)
- 技术路线图: `输入 → 模块1 → 模块2 → 输出`
- 每个关键模块:
  - 解决什么子问题
  - 具体技术点 (结构/损失/训练策略/先验)
- 关键假设列表 + 适用场景
- 理论分析/toy example 的核心结论 (如有)

#### 3. 实验与局限 (Experiments & Limitations)
- 实验设置:
  - 数据集 & 任务
  - 模型规模 & 训练细节 (batch size, steps, 超参)
  - 关键 baseline (3 个左右)
- 收益分析: 哪些指标有明显收益
- 局限分析:
  - 作者承认的
  - 你补充的 (评测偏窄、数据泄漏、domain shift 等)

#### 4. 可复现性与成本 (Reproducibility)
- 代码/数据状态: 开源程度、具体资源
- 复现关键细节: 模型大小、显存需求、训练时间
- 成本估算:
  - GPU 小时
  - 复现难度 (本科高年级 / 研究生 / 资深工程师)

#### 5. 学习价值与实践建议 (For Me) ⭐
> 这是区别于 Referee Report 的**核心差异化模块**

- **技术学习点** (2-3 个): 值得学的技巧/策略/分析方法
- **工程实践建议**:
  - 适合什么具体场景试验
  - 落地需要额外注意/修改的技术细节 (2-3 个)
- **综合判断** (1 句):
  - `建议亲自复现` / `建议只读懂思路不必复现` / `建议当作背景阅读`
  - + 理由

## 两种模式的对比

| 维度 | Referee Report | Technical Learning |
|------|----------------|-------------------|
| 核心问题 | 这篇论文好不好？ | 我能学到什么？ |
| 输出风格 | Checklist 打勾评分 | 结构化叙述 + 具体建议 |
| 特色模块 | Harm Risk / Anti-use-cases | For Me (学习点 + 落地建议) |
| 适用场景 | 调研、写综述 | 深度阅读、工程复用 |
| 信息密度 | 高密度压缩 | 详细展开 |

---

## 扩展思路

1. **领域定制**: 在 System Prompt 末尾加上 "尤其关注 XXX 相关内容"
   - 例如: "尤其关注对齐/安全相关的实验和讨论"
   
2. **模板配置化**: 把 System Prompt 存入 `config/review_templates.yaml`，允许用户自定义

3. **批量对比模式**: 同一模板评 3-5 篇论文，生成对比表格
